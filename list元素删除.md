### list元素删除
* 经常有这么一个场景，我们需要删除一个list里面不符合条件的元素，但python里list对象删除元素这个操作，看着简单，坑却很深。看个例子：

		>>> aa = [1,2,3,4,5,6]
		>>> for a in aa:
		...     if a%2 == 0:
		...             aa.remove(a)
		...
		>>>
		>>> aa
		[1, 3, 5]
* 这种情况下，删除操作是正常的。

		>>> aa = [2,4,5,6]
		>>> for a in aa:
		...     if a%2 == 0:
		...             aa.remove(a)
		...
		>>> aa
		[4, 5]
* 这个时候，我们期望的aa结果是[5]，为什么是[4,5]了？
* list对象，在内存中是个链表的结构，当list对象删除一个元素后，python为了保证list对象依然是连续的，不会断裂，会把被删除的的前一个元素和后一个元素连接起来，也就是说，所有元素向前移动了一位，但是迭代器并没有改变。
* 对于第一个例子，当第一个元素2被删除后，aa变成了[1,3,4,5,6]，但是迭代器的位置指在了元素4上，所以，元素3被跳过了。以此看下来，第一个例子正好被跳过的元素是不需要删除的，所以第一个例子结果正确，第二个例子中，元素4被跳过了，所以返回的结果不符合预期。
